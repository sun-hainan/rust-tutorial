## 1. Vec<T> vs [T; N]

### 生活类比 🎯

> Vec 就像一条可以伸缩的橡皮筋——初始长度固定，但可以拉伸（push）或收缩（pop）。固定数组就像一根硬铁丝——长度永远不变。

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│            Vec<T> 容量增长示意                            │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  push 1 → [1]              cap=1                        │
│  push 2 → [1, 2]           cap=2 (触发扩容)             │
│  push 3 → [1, 2, 3]        cap=4 (容量翻倍)             │
│  push 4 → [1, 2, 3, 4]    cap=4                        │
│  push 5 → [1, 2, 3, 4, 5] cap=8 (再次扩容翻倍)         │
│                                                          │
│  典型容量增长: 1 → 2 → 4 → 8 → 16 → 32 ...            │
│  每次扩容需要: 分配新内存 + 拷贝数据                     │
│  建议: 用 Vec::with_capacity() 预分配已知大小            │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn vec_vs_array() {
    // 固定数组
    let arr: [i32; 5] = [1, 2, 3, 4, 5];

    // 动态数组
    let mut vec = Vec::new();
    vec.push(1);
    vec.push(2);
    vec.push(3);

    // with_capacity 预分配
    let mut vec2 = Vec::with_capacity(100);
    println!("vec2 cap = {}", vec2.capacity()); // 100

    // 容量增长演示
    let mut caps = vec![];
    let mut growing = vec![];
    for i in 0..10 {
        growing.push(i);
        caps.push(growing.capacity());
    }
    println!("capacity growth: {:?}", caps);
}
```

## 2. HashMap<K, V>

### 生活类比 🎯

> HashMap 就像图书馆的索引卡片柜——不管书有多少，找书只需要几秒钟。管理员（哈希函数）根据书名计算出柜子编号，直接去那个柜子拿。

### 概念讲解

HashMap 使用哈希表实现 O(1) 平均查找/插入/删除。

**哈希表核心原理：**
1. 哈希函数：将 key 映射到桶索引
2. 冲突处理：链地址法（separate chaining）
3. 扩容：当负载因子超过阈值（约 0.75），rehash 扩容

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│              HashMap 内部结构示意                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  key = "Alice", hash(key) = 3 → 桶[3]                  │
│  key = "Bob",   hash(key) = 1 → 桶[1]                  │
│  key = "Carol", hash(key) = 3 → 桶[3]（冲突，链表）     │
│                                                          │
│  ┌────┬────┬────┬────┬────┬────┬────┬────┐            │
│  │ 0  │ 1  │ 2  │ 3  │ 4  │ 5  │ 6  │ 7  │  桶数组   │
│  └──┬─┴────┴────┴──┬─┴────┴────┴────┴────┘            │
│     │              │                                     │
│     ▼              ▼                                     │
│  [Bob:85]    [Alice:100]                               │
│                └──→ [Carol:92]                          │
│                                                          │
│  查找 "Carol": hash → 桶[3] → 遍历链表 → 找到          │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn hashmap_usage() {
    use std::collections::HashMap;
    let mut scores = HashMap::new();

    // 插入
    scores.insert("Alice", 100);
    scores.insert("Bob", 85);

    // 查找
    match scores.get("Alice") {
        Some(score) => println!("Alice's score = {}", score),
        None => println!("Alice not found"),
    }

    // 更新
    scores.entry("Alice").and_modify(|e| *e += 10);

    // or_insert（插入或获取）
    let counter = scores.entry("Charlie").or_insert(0);
    *counter += 1;

    // 遍历
    for (name, score) in &scores {
        println!("{}: {}", name, score);
    }
}
```

## 3. HashSet<T>

### 概念讲解

HashSet 是 HashMap<T, ()> 的封装，保证元素唯一。支持集合运算：union, intersection, difference。

### 代码示例

```rust
fn hashset_usage() {
    use std::collections::HashSet;

    let mut set = HashSet::new();
    set.insert("apple");
    set.insert("banana");
    set.insert("apple"); // 重复，不会添加
    println!("set size = {}", set.len()); // 2

    // 集合运算
    let a: HashSet<i32> = [1, 2, 3, 4].iter().cloned().collect();
    let b: HashSet<i32> = [3, 4, 5, 6].iter().cloned().collect();

    println!("a ∪ b = {:?}", a.union(&b).collect::<HashSet<_>>());
    println!("a ∩ b = {:?}", a.intersection(&b).collect::<HashSet<_>>());
    println!("a - b = {:?}", a.difference(&b).collect::<HashSet<_>>());

    // 去重
    let nums = vec![1, 2, 2, 3, 3, 3, 4, 4, 5];
    let unique: HashSet<i32> = nums.into_iter().collect();
    println!("unique: {:?}", unique);
}
```

## 4. VecDeque<T>（双端队列）

### 生活类比 🎯

> VecDeque 就像一个双向地铁站——你可以在任意一端上车（push）或下车（pop）。比 Vec 只允许一端操作更灵活。

### 适用场景

- BFS 遍历
- 滑动窗口
- 生产者-消费者模式

### 代码示例

```rust
fn deque_usage() {
    use std::collections::VecDeque;

    let mut deque = VecDeque::new();
    deque.push_back(1);
    deque.push_back(2);
    deque.push_front(0);  // 从前端入队
    deque.push_front(-1);

    println!("deque: {:?}", deque);

    let front = deque.pop_front();
    let back = deque.pop_back();
    println!("pop_front = {:?}, pop_back = {:?}", front, back);
}
```

## 5. BinaryHeap<T>（优先队列）

### 概念讲解

BinaryHeap 是**最大堆**，根节点是最大元素。

### 代码示例

```rust
fn binary_heap_usage() {
    use std::collections::BinaryHeap;
    use std::cmp::Reverse;

    let mut heap = BinaryHeap::new();
    heap.push(3);
    heap.push(1);
    heap.push(4);

    println!("peek max = {:?}", heap.peek()); // Some(4)

    // 出堆（最大优先）
    while let Some(top) = heap.pop() {
        print!("{} ", top);
    }
    println!();

    // 最小堆（用 Reverse）
    let mut min_heap: BinaryHeap<Reverse<i32>> = BinaryHeap::new();
    min_heap.push(Reverse(3));
    min_heap.push(Reverse(1));
    min_heap.push(Reverse(4));

    while let Some(Reverse(v)) = min_heap.pop() {
        print!("{} ", v);
    }
}
```

## 本章小结

| 集合 | 特点 | 复杂度 |
|------|------|--------|
| Vec<T> | 动态数组，尾部操作 O(1) | 尾部 push/pop O(1)，随机访问 O(1) |
| HashMap<K,V> | 哈希表，键值对 | 平均 O(1) 查找/插入 |
| HashSet<T> | 哈希集合，元素唯一 | 同 HashMap |
| VecDeque<T> | 双端队列 | 两端 O(1) |
| BinaryHeap<T> | 最大堆 | O(log n) 插入/删除 |
| LinkedList<T> | 双向链表 | 随机访问 O(n) |
| BTreeMap | 有序映射 | O(log n)，支持范围查询 |
