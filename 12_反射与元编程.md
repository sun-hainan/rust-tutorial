## 1. Rust 的反射能力

### 生活类比 🎯

> Rust 没有 Java 那样的完整反射——因为 Rust 设计哲学强调"静态确定"，让编译器做更多检查更安全。但 Rust 提供了"有限反射"：Any trait 可以让你在运行时问"这个值是什么类型？"

### 概念讲解

Rust 的反射能力非常有限：
- `std::any::Any`：类型擦除
- `TypeId`：类型标识
- `std::mem::size_of`：编译期信息
- `proc_macro`：编译时代码生成

### 代码示例

```rust
fn reflection_basics() {
    // Any trait：类型擦除
    let boxed: Box<dyn Any> = Box::new(42i32);
    if let Some(num) = boxed.downcast_ref::<i32>() {
        println!("downcasted to i32: {}", num);
    }

    // TypeId 获取类型标识
    println!("i32 type id = {:?}", TypeId::of::<i32>());

    // type_name 获取类型名字（运行时）
    println!("type of 42 = {}", std::any::type_name_of_val(&42));

    // size_of 获取类型大小
    println!("size of i32 = {}", std::mem::size_of::<i32>());
    println!("size of f64 = {}", std::mem::size_of::<f64>());
}
```

## 2. Any trait

### 代码示例

```rust
fn any_trait_usage() {
    // 存储不同类型
    let mut objects: Vec<Box<dyn Any>> = vec![];
    objects.push(Box::new(42_i32));
    objects.push(Box::new("hello".to_string()));
    objects.push(Box::new(vec![1, 2, 3]));

    for obj in objects {
        if let Some(i) = obj.downcast_ref::<i32>() {
            println!("i32: {}", i);
        } else if let Some(s) = obj.downcast_ref::<String>() {
            println!("String: {}", s);
        } else if let Some(v) = obj.downcast_ref::<Vec<i32>>() {
            println!("Vec<i32>: {:?}", v);
        }
    }

    // downcast（获取所有权）
    let obj: Box<dyn Any> = Box::new(100_i32);
    let recovered = obj.downcast::<i32>().unwrap();
    println!("recovered: {}", *recovered);
}
```

## 3. 宏的三种类型

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│              Rust 宏的三种类型                           │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  1. 声明宏（declarative macro）                         │
│     macro_rules! vec_eq {                              │
│         ($($e:expr),*) => {{                           │
│             let mut v = Vec::new();                    │
│             $(v.push($e);)*                            │
│             v                                          │
│         }};                                            │
│     }                                                  │
│                                                          │
│  2. 函数宏（function-like macro）                       │
│     println!("{}", x);  ← 像函数一样调用                │
│     panic!("error: {}", msg);                          │
│                                                          │
│  3. 派生宏（derive macro）                              │
│     #[derive(Debug, Clone)]                            │
│     struct User { name: String }                       │
│     → 编译器生成 Debug::fmt 和 Clone::clone            │
│                                                          │
│  4. 属性宏（attribute macro）                           │
│     #[route(GET, "/")]                                  │
│     fn index() { }                                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn macro_types() {
    // 声明宏（macro_rules!）
    macro_rules! vec_eq {
        ($($e:expr),*) => {{
            let mut v = Vec::new();
            $(v.push($e);)*
            v
        }};
    }
    let v = vec_eq![1, 2, 3];
    println!("vec_eq: {:?}", v);

    // 更复杂的声明宏
    macro_rules! hash_map {
        ($($k:expr => $v:expr),*) => {{
            let mut map = std::collections::HashMap::new();
            $(map.insert($k, $v);)*
            map
        }};
    }
    let map = hash_map!["a" => 1, "b" => 2];
    println!("hash_map: {:?}", map);
}
```

## 4. PhantomData

### 生活类比 🎯

> PhantomData 就像一个"隐形标签"——它不占用任何空间（ZST），但编译器会因为有这个标签而记住"这个结构体拥有某种类型"。在 unsafe 编程和泛型设计中非常有用。

### 代码示例

```rust
fn zst_phantomdata() {
    // ZST 示例
    struct NoData;
    println!("size of NoData = {}", std::mem::size_of::<NoData>()); // 0

    // PhantomData 用途：标记泛型
    struct TypedVec<T> {
        data: Vec<i32>,
        _phantom: std::marker::PhantomData<T>,
    }

    let tv: TypedVec<String> = TypedVec {
        data: vec![1, 2, 3],
        _phantom: std::marker::PhantomData,
    };
    println!("TypedVec<String> size = {}", std::mem::size_of::<TypedVec<String>>());
    println!("data size = {}", std::mem::size_of::<Vec<i32>>());
}
```

## 5. const 泛型

### 代码示例

```rust
fn const_generics() {
    // 固定大小数组
    fn print_array<T: std::fmt::Debug, const N: usize>(arr: [T; N]) {
        println!("{:?}", arr);
    }

    print_array([1, 2, 3]);
    print_array(['a', 'b', 'c', 'd']);

    // const 泛型约束
    struct Matrix<T, const ROWS: usize, const COLS: usize> {
        data: [[T; COLS]; ROWS],
    }

    impl<T: Copy + Default, const ROWS: usize, const COLS: usize> Matrix<T, ROWS, COLS> {
        fn new() -> Self {
            Matrix { data: [[T::default(); COLS]; ROWS] }
        }
    }

    let m: Matrix<i32, 2, 3> = Matrix::new();
    println!("2x3 matrix created");
}
```

## 本章小结

| 特性 | Rust 实现 |
|------|-----------|
| 反射 | 有限（Any / TypeId） |
| 声明宏 | `macro_rules!` |
| 派生宏 | `#[derive(...)]` |
| PhantomData | 类型标记，不占空间 |
| const 泛型 | 泛型参数可以是常量 |
| 动态分发 | HashMap + Box<dyn Fn> |
