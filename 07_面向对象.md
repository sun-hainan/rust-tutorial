## 1. 封装：可见性规则

### 生活类比 🎯

> Rust 的可见性就像一栋公寓楼的访问权限——公共区域谁都能进（pub），但私人房间需要钥匙（private）。你甚至可以限制到同一 crate 或父模块内可见。

### 概念讲解

Rust 的可见性修饰符：
- `pub`：公开
- `pub(crate)`：crate 内可见
- `pub(super)`：父模块可见
- 默认（无修饰）：private（本模块内）

### 代码示例

```rust
mod inner {
    pub struct Counter { count: u32 } // pub 对外，但字段 private
    impl Counter {
        pub fn new() -> Self { Counter { count: 0 } }
        pub fn increment(&mut self) { self.count += 1; }
        pub fn get(&self) -> u32 { self.count }
        // private 方法：只能内部调用
        fn private_helper(&self) -> u32 { self.count * 2 }
    }
}

let mut c = inner::Counter::new();
c.increment();
println!("counter = {}", c.get());
// c.count = 10; // ❌ 编译错误：count 是 private
```

## 2. 组合优于继承

### 生活类比 🎯

> Rust 没有继承，因为继承就像"买了套房就继承所有装修"——一旦父类变了，所有子类都得跟着变，很容易出问题。Rust 更推荐"组合"——学生"有"名字，而不是学生"是"名字。

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│          继承 vs 组合                                   │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  继承（❌ Rust 不支持）：                                │
│           ┌─────────┐                                  │
│           │ Animal  │                                  │
│           └────┬────┘                                  │
│         ┌──────┴──────┐                                │
│         ▼             ▼                                │
│     ┌────────┐    ┌────────┐                           │
│     │  Dog   │    │  Cat   │                           │
│     └────────┘    └────────┘                           │
│                                                          │
│  组合（✅ Rust 推荐）：                                  │
│    ┌─────────────┐                                      │
│    │  Student   │                                      │
│    │─────────────│                                      │
│    │ name: Name │────→ Name 结构体（has-a）           │
│    │ grade: u8  │                                      │
│    └─────────────┘                                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn composition_over_inheritance() {
    // 组合示例：学生"有"名字
    #[derive(Debug)]
    struct Name { first: String, last: String }
    #[derive(Debug)]
    struct Student { name: Name, grade: u8 }

    let student = Student {
        name: Name { first: "张".into(), last: "三".into() },
        grade: 90,
    };
    println!("{:?}", student);

    // 嵌套结构体模拟"继承"
    #[derive(Debug)]
    struct Base { x: i32 }
    #[derive(Debug)]
    struct Extended { base: Base, y: i32 }

    let ext = Extended { base: Base { x: 1 }, y: 2 };
    println!("base.x = {}", ext.base.x);
}
```

## 3. Trait 多态

### 生活类比 🎯

> Trait 就像"演员合同"——不管你是人还是机器人，只要你签了 Speak 合同，就得会说话（实现 speak 方法）。合同就是规范，签约就是实现 trait。

### 代码示例

```rust
fn trait_polymorphism() {
    trait Drawable {
        fn draw(&self);
        fn area(&self) -> f64; // 必须实现
    }

    struct Circle { radius: f64 }
    struct Rectangle { width: f64, height: f64 }

    impl Drawable for Circle {
        fn draw(&self) { println!("Circle(r={:.2})", self.radius); }
        fn area(&self) -> f64 { std::f64::consts::PI * self.radius * self.radius }
    }
    impl Drawable for Rectangle {
        fn draw(&self) { println!("Rectangle({:.2}x{:.2})", self.width, self.height); }
        fn area(&self) -> f64 { self.width * self.height }
    }

    let shapes: Vec<Box<dyn Drawable>> = vec![
        Box::new(Circle { radius: 1.0 }),
        Box::new(Rectangle { width: 2.0, height: 3.0 }),
    ];

    for shape in &shapes {
        shape.draw();
        println!("  area = {:.2}", shape.area());
    }
}
```

## 4. 静态分发 vs 动态分发

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│          静态分发（泛型） vs 动态分发（dyn Trait）       │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  静态分发（编译时确定类型）：                            │
│  ┌─────────────────────────────────┐                     │
│  │ fn print_drawable<D: Drawable> │                     │
│  │     (d: &D) { d.draw() }       │                     │
│  │ 编译器为 Circle 生成:            │                     │
│  │ fn print_drawable_circle(...)   │                     │
│  │ 编译器为 Rectangle 生成:         │                     │
│  │ fn print_drawable_rect(...)     │                     │
│  └─────────────────────────────────┘                     │
│  ✅ 性能好，无虚表查找                                     │
│  ❌ 无法存储异构集合                                      │
│                                                          │
│  动态分发（运行时确定类型）：                            │
│  ┌─────────────────────────────────┐                     │
│  │ fn print_any(d: &dyn Drawable) │                     │
│  │     d.draw()  ← 虚表查找        │                     │
│  └─────────────────────────────────┘                     │
│  ✅ 可存异构集合，✅ 编译产物小                          │
│  ❌ 虚表查找开销                                         │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn static_vs_dynamic_dispatch() {
    trait Speak { fn speak(&self) -> String; }
    struct Dog; struct Cat; struct Cow;
    impl Speak for Dog { fn speak(&self) -> String { "汪汪".into() } }
    impl Speak for Cat { fn speak(&self) -> String { "喵".into() } }
    impl Speak for Cow { fn speak(&self) -> String { "哞".into() } }

    // 动态分发：trait 对象
    let animals: Vec<Box<dyn Speak>> = vec![
        Box::new(Dog), Box::new(Cat), Box::new(Cow),
    ];
    for animal in &animals {
        print!("{} ", animal.speak());
    }

    // 静态分发：泛型
    fn make_speak_all<T: Speak>(animals: &[T]) -> String {
        let mut result = String::new();
        for animal in animals { result.push_str(&animal.speak()); result.push(' '); }
        result
    }
    let dogs = [Dog, Dog];
    println!("静态分发 dogs: {}", make_speak_all(&dogs));
}
```

## 本章小结

| 特性 | Rust 实现 |
|------|-----------|
| 封装 | pub/private/pub(crate)/pub(super) |
| 继承 | 无，用 Trait + 组合替代 |
| 多态 | Trait + impl Trait for Type |
| 静态分发 | 泛型 `<T: Trait>`，编译时确定 |
| 动态分发 | `Box<dyn Trait>`，运行时虚表查找 |
| 关联类型 | trait 中用 `type Item`，impl 时确定 |
