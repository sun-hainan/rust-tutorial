## 1. if/else 表达式

### 生活类比 🎯

> Rust 的 if/else 是"表达式"不是"语句"——就像数学里的分段函数，每个分支都要返回同一个"类型"的值，最终得到一个确定结果。

### 概念讲解

`if/else` 在 Rust 中是表达式，可以返回值。所有分支必须返回相同类型。

编译器视角：if 表达式被翻译成条件跳转指令（brif），两个分支最终汇合到同一个 phi 节点（SSA 形式）。

### 代码示例

```rust
fn if_else_expressions() {
    let number = 7;

    // 基本 if/else
    if number % 2 == 0 {
        println!("{} 是偶数", number);
    } else {
        println!("{} 是奇数", number);
    }

    // if 表达式（所有分支类型必须一致）
    let label = if number % 2 == 0 { "even" } else { "odd" };
    println!("number is {}", label);

    // 多分支
    let score = 85;
    let grade = if score >= 90 {
        "A".to_string()
    } else if score >= 80 {
        "B".to_string()
    } else {
        "D".to_string()
    };
    println!("score {} → grade {}", score, grade);
}
```

## 2. match 模式匹配

### 生活类比 🎯

> match 就像智能开关——不只是开/关，还可以根据输入的值、范围、类型来匹配不同的动作，而且编译器会确保你没有漏掉任何情况。

### 概念讲解

Rust 没有 switch，而是用 match（模式匹配）。相比 C/Java switch，match 强大得多：

- 匹配字面量、范围、枚举、结构体、元组
- **穷尽检查**（exhaustiveness checking）——编译器确保所有情况被覆盖
- 每个分支可以有绑定（destructuring）
- 分支 guard（`if` 条件）

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│                    match 穷尽检查示意                    │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  enum Color { Red, Green, Blue }                        │
│                                                          │
│  let color = Color::Red;                                │
│  match color {                                          │
│      Color::Red   => println!("红色"),                  │
│      Color::Green => println!("绿色"),  ← 漏掉 Blue     │
│      // ❌ 编译器报错：non-exhaustive patterns!        │
│  }                                                      │
│                                                          │
│  match 必须覆盖所有枚举变体，或者用 _ 兜底               │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn match_expressions() {
    let num = 3;

    match num {
        1 => println!("one"),
        2 => println!("two"),
        3 => println!("three"),
        _ => println!("other"), // 通配符
    }

    // 范围匹配
    let score = 85;
    let grade = match score {
        90..=100 => "A",
        80..=89  => "B",
        70..=79  => "C",
        60..=69  => "D",
        _        => "F",
    };

    // 匹配枚举
    enum Direction { North, South, East, West }
    fn get_direction_label(dir: Direction) -> &'static str {
        match dir {
            Direction::North => "北",
            Direction::South => "南",
            Direction::East  => "东",
            Direction::West  => "西",
        }
    }

    // 匹配时绑定变量 + guard
    let msg = Some(42);
    match msg {
        Some(n) if n % 2 == 0 => println!("偶数: {}", n),
        Some(n) if n % 2 != 0 => println!("奇数: {}", n),
        None => println!("空值"),
        _ => println!("其他"),
    }
}
```

## 3. 循环：loop / while / for

### 生活类比 🎯

> 三种循环就像不同的跑步方式：loop 是绕圈跑直到 break；while 是教练喊"只要还没到终点就继续跑"；for 是知道自己要跑多少圈。

### 概念讲解

Rust 三种循环：
- `loop`：无限循环，需手动 break
- `while`：条件循环
- `for`：迭代循环（最常用）

### 代码示例

```rust
fn loop_types() {
    // loop（可带返回值）
    let result = loop {
        let mut counter = 0;
        counter += 1;
        if counter == 10 {
            break counter * 2; // loop 可以有返回值
        }
    };
    println!("loop result = {}", result);

    // while 循环
    let mut n = 0;
    while n < 5 {
        print!("{} ", n);
        n += 1;
    }
    println!();

    // for 遍历（最常用）
    for i in 0..5 {
        print!("{} ", i);
    }
    println!();

    // for 遍历 HashMap
    use std::collections::HashMap;
    let mut scores = HashMap::new();
    scores.insert("Alice", 100);
    scores.insert("Bob", 85);

    for (name, score) in &scores {
        println!("{}: {}", name, score);
    }
}
```

## 4. break / continue 与标签

### 代码示例

```rust
fn break_continue_labels() {
    // break 带值
    let result = loop {
        break 42;
    };
    println!("break with value: {}", result);

    // 标签用于多层循环
    'outer: for i in 1..4 {
        for j in 1..4 {
            if i == 2 && j == 2 {
                println!("break at i={} j={}", i, j);
                break 'outer; // 跳出外层循环
            }
        }
    }
}
```

## 5. 错误处理：panic / Result / Option

### 生活类比 🎯

> Rust 把错误分成两类：一类是"程序 Bug"（panic），比如数组越界；另一类是"可预见的失败"（Result），比如文件不存在。编译器强制你处理所有可预见的错误。

### 代码示例

```rust
fn error_handling() {
    // Result 基本用法
    fn divide(a: i32, b: i32) -> Result<i32, String> {
        if b == 0 {
            Err("除数不能为零".to_string())
        } else {
            Ok(a / b)
        }
    }

    let result = divide(10, 2);
    match result {
        Ok(v)  => println!("10 / 2 = {}", v),
        Err(e) => println!("错误: {}", e),
    }

    // ? 运算符（早期返回）
    fn parse_and_double(s: &str) -> Result<i32, std::num::ParseIntError> {
        let num: i32 = s.trim().parse::<i32>()?;
        Ok(num * 2)
    }
}
```

## 6. 迭代器链（函数式风格）

### 代码示例

```rust
fn iterator_pipeline() {
    // 计算 1..100 中 3 的倍数的和
    let sum: i32 = (1..100)
        .filter(|x| x % 3 == 0)
        .sum();
    println!("1..100 中 3 的倍数的和 = {}", sum);

    // 找出第一个符合条件的元素
    let first = (1..)
        .filter(|x| x * x > 1000)
        .find(|x| *x % 7 == 0);
    println!("第一个平方>1000且被7整除的数 = {:?}", first);

    // fold（折叠）
    let product = (1..=5).fold(1, |acc, x| acc * x);
    println!("1*2*3*4*5 = {}", product); // 120
}
```

## 本章小结

| 特性 | Rust 实现 |
|------|-----------|
| if/else | 表达式，可返回值，穷尽检查 |
| match | 模式匹配，支持范围/枚举/guard |
| 循环 | loop/while/for |
| break | 可返回值，标签跳出外层 |
| 错误处理 | panic（不可恢复）/ Result（可恢复） |
| 迭代器 | map/filter/fold 链式调用 |
