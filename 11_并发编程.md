## 1. 线程创建与 Arc

### 生活类比 🎯

> 线程就像一条新的生产线——主生产线（main thread）启动另一条（spawn）。但问题是：这条新生产线需要的原材料（数据）怎么送来？Arc 就像一个"共享仓库许可证"，让多条生产线可以访问同一个仓库。

### 概念讲解

Rust 标准库提供 `std::thread`，使用 `spawn` 创建线程。

**线程通信方式：**
1. 共享内存（Mutex/RwLock）
2. 消息传递（mpsc channel）

`Arc`（原子引用计数）：实现多线程共享所有权。

### 代码示例

```rust
fn thread_creation() {
    // 基本线程创建
    let handle = thread::spawn(|| {
        println!("子线程执行中...");
        42
    });

    let result = handle.join().unwrap();
    println!("子线程返回: {}", result);

    // 多线程
    let handles: Vec<_> = (0..4).map(|i| {
        thread::spawn(move || {
            println!("线程 {} 执行", i);
            i * 2
        })
    }).collect();

    for h in handles {
        let r = h.join().unwrap();
        println!("线程返回: {}", r);
    }

    // 线程间共享数据
    let shared = Arc::new(vec![1, 2, 3]);
    let shared2 = Arc::clone(&shared);
    thread::spawn(move || {
        println!("shared2 in thread: {:?}", shared2);
    }).join().unwrap();
}
```

## 2. Mutex 实现线程安全

### 代码示例

```rust
fn mutex_usage() {
    // Arc + Mutex 实现多线程安全共享
    let counter = Arc::new(Mutex::new(0));

    let mut handles = vec![];
    for i in 0..5 {
        let counter = Arc::clone(&counter);
        let handle = thread::spawn(move || {
            let mut num = counter.lock().unwrap();
            *num += 1;
            println!("线程 {} 增加计数到 {}", i, *num);
        });
        handles.push(handle);
    }

    for h in handles {
        h.join().unwrap();
    }
    println!("最终计数: {}", *counter.lock().unwrap());

    // RwLock（读写锁）：读多写少场景更高效
    let data = Arc::new(RwLock::new(vec![1, 2, 3]));

    // 读操作（多个读者）
    let r1 = Arc::clone(&data);
    let reader1 = thread::spawn(move || {
        let v = r1.read().unwrap();
        println!("reader1: {:?}", v);
    });

    reader1.join().unwrap();
}
```

## 3. 消息传递：mpsc channel

### 生活类比 🎯

> channel 就像工厂的传送带——生产者把产品放上去，消费者从另一端拿走。没有共享内存的争抢，只有有序的传递。

### 代码示例

```rust
fn channel_communication() {
    use std::sync::mpsc;
    use std::time::Duration;

    // 创建 channel
    let (tx, rx) = mpsc::channel();

    // 生产者线程
    let producer = thread::spawn(move || {
        for i in 0..5 {
            tx.send(i).unwrap();
            thread::sleep(Duration::from_millis(100));
        }
    });

    // 主线程接收
    while let Ok(val) = rx.recv() {
        println!("received: {}", val);
    }

    producer.join().unwrap();
    println!("producer finished");

    // 多生产者
    let (tx, rx) = mpsc::channel();
    let tx2 = tx.clone();

    let p1 = thread::spawn(move || {
        for i in 0..3 { tx.send(format!("P1: {}", i)).unwrap(); }
    });

    let p2 = thread::spawn(move || {
        for i in 0..3 { tx2.send(format!("P2: {}", i)).unwrap(); }
    });

    drop(tx); drop(tx2);

    for received in rx {
        println!("got: {}", received);
    }
}
```

## 4. 死锁避免

### 生活类比 🎯

> 死锁就像两个司机在窄桥上相遇——A 等 B 让路，B 等 A 让路，谁也不动。解决方法很简单：固定让行顺序，永远是同一个人先让。

### 死锁四个必要条件

1. **互斥**：资源只能被一个线程持有
2. **占有并等待**：线程持有资源同时等待其他资源
3. **不可抢占**：资源不能被强制夺走
4. **循环等待**：形成线程环形依赖

### 避免策略

- **固定加锁顺序**（防止循环等待）
- **减少锁的持有时间**
- 使用 scoped thread

### 代码示例

```rust
fn deadlock_prevention() {
    // 正确做法：固定顺序
    let a = Arc::new(Mutex::new(1));
    let b = Arc::new(Mutex::new(2));

    let a1 = Arc::clone(&a);
    let b1 = Arc::clone(&b);

    // 线程1：先 a 后 b
    let t1 = thread::spawn(move || {
        let _a = a1.lock().unwrap();
        thread::sleep(Duration::from_millis(10));
        let _b = b1.lock().unwrap();
        println!("thread1: got both locks");
    });

    // 线程2：也是先 a 后 b（固定顺序避免循环）
    let a2 = Arc::clone(&a);
    let b2 = Arc::clone(&b);

    let t2 = thread::spawn(move || {
        let _a = a2.lock().unwrap();
        thread::sleep(Duration::from_millis(10));
        let _b = b2.lock().unwrap();
        println!("thread2: got both locks");
    });

    t1.join().unwrap();
    t2.join().unwrap();
}
```

## 本章小结

| 机制 | 说明 |
|------|------|
| `std::thread::spawn` | 创建 OS 线程 |
| `Arc<T>` | 原子引用计数，多线程共享所有权 |
| `Mutex<T>` | 互斥锁，同一时刻只允许一个线程访问 |
| `RwLock<T>` | 读写锁，多读者单写者 |
| `mpsc::channel` | 多生产者单消费者消息传递 |
| `Condvar` | 条件变量，等待某个条件成立 |
| `Barrier` | 屏障，等待所有线程都到达某点 |
| `Send/Sync` | 标记可跨线程传递/共享的类型 |
