## 1. 路径处理：Path / PathBuf

### 生活类比 🎯

> Path 就像一个地址的"引用"（你可以抄写在纸上给别人看，但地址本身不属于你），而 PathBuf 就像你"拥有"那块地（可以改造）。

### 概念讲解

| 类型 | 说明 |
|------|------|
| `Path` | 切片引用，不可变，跨平台 |
| `PathBuf` | 拥有所有权的字符串，可变 |

### 代码示例

```rust
fn path_handling() {
    use std::path::Path;

    let path = Path::new("src/main.rs");
    println!("path = {:?}", path);

    // PathBuf（拥有所有权）
    let mut path_buf = PathBuf::from("workspace");
    path_buf.push("src");
    path_buf.push("lib");
    println!("path_buf = {:?}", path_buf);

    // 路径组件
    println!("file name = {:?}", path_buf.file_name());
    println!("extension = {:?}", path_buf.extension());
    println!("parent = {:?}", path_buf.parent());

    // 路径存在性检查
    let doc = Path::new("Cargo.toml");
    println!("'Cargo.toml' exists = {}", doc.exists());
    println!("is file = {}", doc.is_file());
    println!("is dir = {}", doc.is_dir());
}
```

## 2. 文件读写：File / BufReader / BufWriter

### 生活类比 🎯

> BufReader/BufWriter 就像带缓冲的公交站——不是来一个人就走，而是一批人集中上车，减少了频繁停靠的开销。大量小读写时，缓冲可提升数十倍性能。

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│         无缓冲 vs 带缓冲 IO 对比                        │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  无缓冲：                                               │
│  每次 read()  → 系统调用 → 等待磁盘 → 返回               │
│  每次 write() → 系统调用 → 等待磁盘 → 返回               │
│  问题：大量小 IO，系统调用开销大                         │
│                                                          │
│  带缓冲：                                               │
│  ┌────────────────────────────────┐                     │
│  │ BufReader:                     │                     │
│  │  read() → 填满缓冲区(4KB)     │                     │
│  │  用户从缓冲区取数据            │                     │
│  │  缓冲区空 → 再填满             │                     │
│  └────────────────────────────────┘                     │
│  ┌────────────────────────────────┐                     │
│  │ BufWriter:                     │                     │
│  │  write() → 写入缓冲区          │                     │
│  │  缓冲区满 → flush() → 磁盘    │                     │
│  └────────────────────────────────┘                     │
│  优势：减少系统调用次数                                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn file_io() {
    use std::fs::{File, OpenOptions};
    use std::io::Write;

    // 创建文件
    let mut file = File::create("test_output.txt").unwrap();
    file.write_all(b"Hello, Rust!\n").unwrap();

    // 读取整个文件
    let content = std::fs::read_to_string("test_output.txt").unwrap();
    println!("file content:\n{}", content);

    // 追加模式
    let mut file = OpenOptions::new()
        .append(true)
        .open("test_output.txt")
        .unwrap();
    file.write_all(b"Appended line\n").unwrap();
}
```

## 3. 命令行参数与标准流

### 代码示例

```rust
fn cli_and_streams() {
    // 命令行参数
    let args: Vec<String> = std::env::args().collect();
    println!("args count = {}", args.len());

    // 格式化输出
    let name = "Alice";
    let age = 30;
    println!("{name} is {age} years old");
    println!("{0} {1} {0}", "Hello", "World");
    println!("{:>10}", "right");     // 右对齐
    println!("{:.<10}", "dotted");  // 左对齐，填充

    // 写格式化的字符串
    let formatted = format!("PI ≈ {:.4}", std::f64::consts::PI);
    println!("formatted: {}", formatted);
}
```

## 4. 错误处理：io::Error

### 代码示例

```rust
fn io_error_handling() {
    use std::io;
    use std::path::Path;

    fn read_config() -> Result<String, io::Error> {
        let path = Path::new("config.json");
        if !path.exists() {
            return Err(io::Error::new(
                io::ErrorKind::NotFound,
                "config.json not found",
            ));
        }
        std::fs::read_to_string(path)
    }

    match read_config() {
        Ok(content) => println!("config: {}", content),
        Err(e) => {
            println!("error: {}", e);
            match e.kind() {
                io::ErrorKind::NotFound => println!("  → 文件不存在"),
                io::ErrorKind::PermissionDenied => println!("  → 权限拒绝"),
                _ => println!("  → 其他错误"),
            }
        }
    }
}
```

## 本章小结

| 组件 | 说明 |
|------|------|
| Path / PathBuf | 跨平台路径处理 |
| File | 文件句柄，实现 Read/Write trait |

| BufWriter | 缓冲写入，批量刷新 |
| serde | 序列化框架，支持 JSON/TOML/Bincode |
| io::ErrorKind | I/O 错误类型枚举 |
