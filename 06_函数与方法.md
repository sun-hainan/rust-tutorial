## 1. 函数基础与调用机制

### 生活类比 🎯

> 函数就像工厂的加工流水线——你把原材料（参数）送进去，工厂按固定流程加工，然后产出产品（返回值）。Rust 的函数调用是"值传递"，要么复制一份材料，要么借用。

### 概念讲解

函数定义：`fn` 关键字，参数有类型标注，返回值用 `-> T`

函数调用：值传递（Copy 类型）或借用（引用类型）

编译器视角：函数调用被翻译成 call 指令，参数通过寄存器或栈传递，返回值通过寄存器或内存返回。

### 代码示例

```rust
fn function_basics() {
    // 基本函数
    fn add(a: i32, b: i32) -> i32 { a + b }
    println!("add(3, 4) = {}", add(3, 4));

    // 多返回值（用元组）
    fn min_max(arr: &[i32]) -> (i32, i32) {
        let min = *arr.iter().min().unwrap_or(&0);
        let max = *arr.iter().max().unwrap_or(&0);
        (min, max)
    }
    let (min, max) = min_max(&[3, 1, 4, 1, 5, 9, 2, 6]);
    println!("min={}, max={}", min, max);

    // 函数作为参数（高阶函数）
    fn apply<F>(f: F, x: i32) -> i32
    where F: Fn(i32) -> i32 {
        f(x)
    }
    let result = apply(|x| x * 2, 21);
    println!("apply double 21 = {}", result);
}
```

## 2. 多态：泛型 vs Trait 对象

### 概念讲解

Rust **不支持**同名不同参数类型的重载（无函数重载）。

多态通过以下方式实现：
- 泛型函数（编译时多态）
- Trait 对象（运行时多态）

### 代码示例

```rust
fn no_overload_but_polymorphism() {
    // 泛型函数：同一函数处理多种类型
    fn largest<T: PartialOrd>(list: &[T]) -> &T {
        let mut largest = &list[0];
        for item in list {
            if item > largest { largest = item; }
        }
        largest
    }

    let nums = vec![34, 50, 25, 100, 65];
    let chars = vec!['y', 'm', 'a', 'q'];
    println!("largest i32: {}", largest(&nums));
    println!("largest char: {}", largest(&chars));
}
```

## 3. 递归函数与尾递归

### 生活类比 🎯

> 递归就像俄罗斯套娃——每打开一个，里面还有一个更小的，直到最小的那一个。尾递归优化就是把"套娃的打开动作"变成"直接展示最里面那个"，省去了中间的开销。

### 代码示例

```rust
fn recursion_and_tail_call() {
    // 普通递归：阶乘
    fn factorial(n: u64) -> u64 {
        if n <= 1 { 1 } else { n * factorial(n - 1) }
    }
    println!("5! = {}", factorial(5)); // 120

    // 尾递归版本：累加器模式
    fn factorial_tail(n: u64) -> u64 {
        fn inner(n: u64, acc: u64) -> u64 {
            if n <= 1 { acc } else { inner(n - 1, n * acc) }
        }
        inner(n, 1)
    }

    // 迭代版本（避免栈溢出）
    fn fibonacci_iter(n: u32) -> u32 {
        if n <= 1 { return n; }
        let (mut a, mut b) = (0, 1);
        for _ in 2..=n {
            let next = a + b;
            a = b;
            b = next;
        }
        b
    }
    println!("fib(10) iter = {}", fibonacci_iter(10));
}
```

## 4. 值传递 vs 引用传递

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│           值传递（Copy）vs 引用传递 vs 移动              │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  Copy 类型（i32）：                                      │
│  let a = 5;                                             │
│  consume(a);        // 复制 a 的值                       │
│  println!("{}", a); // ✅ a 仍然有效                     │
│                                                          │
│  Move 类型（String）：                                   │
│  let s1 = String::from("hello");                        │
│  consume_string(s1);  // s1 的所有权移动到函数内        │
│  println!("{}", s1); // ❌ 编译错误，s1 已无效          │
│                                                          │
│  引用传递（&T）：                                        │
│  let s2 = String::from("world");                         │
│  borrow_string(&s2);  // 借用，不获取所有权            │
│  println!("{}", s2); // ✅ s2 仍然有效                  │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn value_vs_reference() {
    // Copy 类型（i32）
    let a = 5;
    fn consume_i32(x: i32) { println!("consumed i32: {}", x); }
    consume_i32(a);
    println!("a still valid: {}", a); // ✅ i32 是 Copy

    // Move 类型（String）
    let s1 = String::from("hello");
    fn consume_string(s: String) { println!("consumed: {}", s); }
    consume_string(s1);
    // println!("s1 still valid: {}", s1); // ❌ 已移动

    // 引用传递
    let s2 = String::from("world");
    fn borrow_string(s: &String) { println!("borrowed: {}", s); }
    borrow_string(&s2);
    println!("s2 still valid: {}", s2); // ✅ 借用
}
```

## 5. 闭包（Closure）

### 生活类比 🎯

> 闭包就像一个"打包好的任务卡"——它记录了当时的上下文环境（捕获的变量），然后可以随时拿出来执行。Fn 是"只读"的任务卡，FnMut 是"可以涂改"的任务卡，FnOnce 是"用了就销毁"的任务卡。

### 概念讲解

**闭包类型：**
- `Fn`：可以多次调用，不修改环境
- `FnMut`：可变借用环境，可以修改
- `FnOnce`：一次性消费环境

编译器视角：闭包是匿名结构体，实现了对应的 trait。环境通过捕获的方式（move、immutable borrow、mutable borrow）传递给闭包。

### 代码示例

```rust
fn closure_mechanics() {
    // 基本闭包
    let add = |a, b| a + b;
    println!("add(3, 4) = {}", add(3, 4));

    // 捕获环境变量
    let factor = 10;
    let multiply = |x| x * factor;
    println!("multiply(3) = {}", multiply(3));

    // move 闭包（强制获取所有权）
    let data = vec![1, 2, 3];
    let consumer = move || {
        println!("consumed data: {:?}", data);
        // data 在闭包内有效
    };
    consumer();
    // println!("data: {:?}", data); // ❌ 已移动到闭包
}
```

## 本章小结

| 特性 | Rust 实现 |
|------|-----------|
| 函数重载 | 不支持，用泛型或 Trait 对象 |
| 多态 | 泛型（编译时）+ dyn Trait（运行时） |
| 递归 | 支持，尾递归可手动优化 |
| 值传递 | Copy 类型复制，Move 类型转移所有权 |
| 闭包 | `\|x\| expr`，实现 Fn/FnMut/FnOnce |
| 泛型约束 | `where T: Trait` 语法 |
