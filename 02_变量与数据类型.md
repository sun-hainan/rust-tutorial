## 1. 变量声明与不可变性

### 生活类比 🎯

> Rust 的变量默认不可变，就像刻在石头上的字——你想改就得用"mut"特别声明。这不是限制，而是让编译器帮你确保代码的正确性。

### 概念讲解

默认：`let` 声明的变量是不可变的（immutable）。必须显式使用 `mut` 关键字才可变。

编译器视角：不可变变量在编译时就被标记，编译器可以：
- 进行更多优化（如常量折叠、循环不变代码外提）
- 检测到意外的多重写入会报编译错误（数据竞争预防）

### 代码示例（逐行中文注释）

```rust
fn variable_immutability() {
    let x = 5;
    // x = 6; // ❌ 编译错误：cannot assign to immutable variable `x`

    let mut y = 5;
    y = 6; // ✅ 可变变量
    println!("y = {}", y);
}
```

## 2. 所有权（Ownership）机制

### 生活类比 🎯

> 所有权就像一本书——你把书借给别人，自己就没有了。Rust 用"移动"来表达这个概念，确保同一本书永远只有一个人持有。

### 概念讲解

Rust 的核心创新：所有权系统，无 GC 但无数据竞争。

**三条规则：**
1. 每个值有且只有一个所有者
2. 当所有者离开作用域，值被 drop（释放内存）
3. 值的所有权可以移动（move）到另一个变量

机器视角：所有权本质上是对"堆内存分配"的追踪。编译器在编译时计算每个值的生命周期，插入 free() 调用。

### ASCII 图示

```
┌──────────────────────────────────────────────────────────┐
│                    所有权移动示意                          │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  let s1 = String::from("hello");  // s1 拥有字符串      │
│       │                                                   │
│       ▼                                                   │
│  let s2 = s1;  // 所有权从 s1 移动到 s2                  │
│       │                                                   │
│       ▼                                                   │
│  println!("{}", s1); // ❌ s1 已无效，编译错误            │
│       │                                                   │
│       ▼                                                   │
│  println!("{}", s2); // ✅ s2 持有所有权                  │
│                                                          │
└──────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn ownership_demo() {
    let s1 = String::from("hello"); // s1 拥有堆上的字符串数据
    let s2 = s1;                    // 所有权从 s1 移动到 s2
    // println!("{}", s1);          // ❌ 编译错误：s1 的所有权已移动
    println!("s2 = {}", s2);

    // 整数等 Copy 类型不受影响（它们在栈上）
    let a = 5;
    let b = a; // Copy：a 的值被复制，a 仍有效
    println!("a = {}, b = {}", a, b);
}
```

## 3. 借用（Borrowing）与引用

### 生活类比 🎯

> 借用就像去图书馆看书——你只是"借"来翻阅，看完还回去，书的所有权还在原主人那里。`&` 是不动产，"看看就好"；`&mut` 是借来做笔记，用完要还回来。

### 概念讲解

**借用：** 允许不获取所有权而使用某个值。
- `&T` 不可变引用
- `&mut T` 可变引用

**借用规则：**
- 任意时刻：要么有多个不可变引用，要么有唯一的一个可变引用（二选一）
- 引用必须始终有效（无空引用 / 无悬垂引用）

编译器视角：借用检查器（borrow checker）在编译时验证这些规则。

### ASCII 图示

```
┌─────────────────────────────────────────────────────────┐
│                    借用规则示意                           │
├─────────────────────────────────────────────────────────┤
│                                                          │
│   多个 &T (不可变引用)  ✅ OK                           │
│          │  │  │                                         │
│          ▼  ▼  ▼                                         │
│         [值] ←── 三个人同时看书                           │
│                                                          │
│   一个 &mut T (可变引用)  ✅ OK                         │
│          │                                               │
│          ▼                                               │
│     [值] ←── 一个人借来做笔记                            │
│                                                          │
│   &T + &mut T  ❌ 错误                                   │
│     │        │                                           │
│     ▼        ▼                                           │
│    [值] ←── 同时看书和做笔记？冲突！                      │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 代码示例

```rust
fn borrowing_demo() {
    let mut s = String::from("hello");

    // 不可变借用
    let len = calculate_length(&s);
    println!("'{}' 的长度是 {}", s, len);

    // 可变借用
    append_world(&mut s);
    println!("追加后: {}", s);
}

fn calculate_length(s: &String) -> usize {
    s.len() // 借用，不获取所有权
}

fn append_world(s: &mut String) {
    s.push_str(", world");
}
```

## 4. 作用域与生命周期

### 生活类比 🎯

> 生命周期就像租房子——你签的租约决定了你在这房子里能住多久。引用也一样，编译器要确保"引用"不会在"房子"被拆之后还存在。

### 概念讲解

**生命周期：** 引用保持有效的时间范围。编译器用 `'a` 等生命周期参数来追踪引用的有效范围。

机器视角：每个引用包含一个指针和一个"租期"信息，编译器在编译时计算引用的有效范围，在超出范围时禁止使用。

### 代码示例

```rust
fn lifetime_scope_demo() {
    let outer_var = String::from("outer");

    {
        let inner_var = String::from("inner");
        // inner_var 的生命周期只在内部块内
        let ref1 = &inner_var;
        println!("inner ref = {}", ref1);
        // 块结束时 inner_var drop，ref1 无效
    }
    // outer_var 仍有效（生命周期更长）
    println!("outer ref = {}", outer_var);

    // 生命周期参数示例
    let longest;
    {
        let s1 = String::from("short");
        let s2 = String::from("very long string");
        longest = longest_string(&s1, &s2);
        println!("longest = {}", longest);
    }
}

// 生命周期标注：告诉编译器两个引用的生命周期关系
fn longest_string<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    if s1.len() > s2.len() { s1 } else { s2 }
}
```

## 5. 基本数据类型

### 概念讲解

| 类型 | 说明 |
|------|------|
| 整数 | i8, i16, i32, i64, i128, isize / u8, u16, u32, u64, u128, usize |
| 浮点 | f32, f64 |
| 布尔 | bool（true / false） |
| 字符 | char（Unicode Scalar Value，4字节） |
| 单元 | ()（表示无返回值或空值） |

### 代码示例

```rust
fn primitive_types() {
    // 整数
    let decimal = 42;          // i32 默认
    let hex_val = 0xFF;       // 十六进制
    let octal = 0o77;        // 八进制
    let binary = 0b1010;      // 二进制
    let byte = b'A';          // u8 字面量（字节）

    // 浮点
    let pi: f64 = 3.141592653589793;

    // 布尔
    let is_rust_fun: bool = true;

    // 字符（Unicode）
    let emoji: char = '🚀';
    println!("emoji = {}", emoji);
    println!("char size: {} bytes", std::mem::size_of::<char>()); // 4
}
```

## 6. Option 替代 null

### 生活类比 🎯

> null 就像"空盘子"——你不知道是没有菜，还是盘子坏了。Rust 的 Option<T> 明确告诉你：要么有值（Some），要么没有（None）。

### 概念讲解

Option<T> 是 Rust 处理"可能无值"的惯用方式：

```rust
enum Option<T> { Some(T), None }
```

设计理念：
- Tony Hoare 发明了 null，称其为"billion dollar mistake"
- null 引用导致运行时错误（NullPointerException）
- Rust 在编译时消除 null，通过类型系统确保"无值"状态必须被处理

### 代码示例

```rust
fn option_handling() {
    let some_value: Option<i32> = Some(42);
    let no_value: Option<i32> = None;

    match some_value {
        Some(v) => println!("got value: {}", v),
        None => println!("no value"),
    }

    // map / unwrap_or
    let result: Option<i32> = Some(10);
    let doubled = result.map(|x| x * 2); // Some(20)
    let fallback = result.unwrap_or(0);  // 10
    println!("doubled = {:?}, fallback = {}", doubled, fallback);
}
```

## 本章小结

| 特性 | Rust 行为 |
|------|-----------|
| 变量默认 | 不可变，mut 可变 |
| 所有权 | 每个值唯一所有者，move 语义 |
| 借用 | `&` 不可变，`&mut` 可变，无空引用 |
| 空值处理 | Option<T> 替代 null，编译时确保处理 |
| 基本类型 | i32/u32/f64/bool/char |
| 复合类型 | tuple/struct/enum |
